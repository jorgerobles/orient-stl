# 02-01-PLAN: three.js Viewport

<objective>
Three.js scene with OrbitControls displaying the loaded model at the current candidate
orientation. Next/prev navigation cycles through ranked candidates, updating the mesh
quaternion and the info readout. Camera auto-frames the model on load.
</objective>

## Must-haves

1. `Viewport` class owning scene/camera/renderer/controls
2. `loadModel(positions, faceNormals)` builds BufferGeometry, frames camera
3. `showCandidate(quaternion)` applies orientation, lifts model onto build plate
4. Prev/next buttons + keyboard nav cycle `candidates[currentIndex]`
5. Candidate info readout: rank, score, height, stability

## Extras landed beyond plan

- Build plate: `GridHelper` + translucent plane at y=0
- Overhang heatmap: per-face vertex colors from world-space normal vs critical angle
- Centroid pivot: modelGroup offset = `-centroid` (center-of-mass rotation)
- Critical angle slider re-colors heatmap live
