# 02-02-PLAN: Yaw Control

<objective>
User adjusts yaw (rotation around the candidate's up-axis) via a circular drag dial
with magnetic snap to geometry-aligned angles (rotating-calipers bbox minima) and a
numeric input. "Reset to auto" restores the default bbox-minimizing yaw from
`computeDefaultYaw`.
</objective>

## Must-haves

1. Circular dial: drag around a circle to set yaw angle
2. Snap-to-geometry: magnetic pull to hull-edge-aligned angles from rotating calipers
3. Numeric input field for precise yaw
4. "Reset to auto" restores `computeDefaultYaw` quaternion
5. Yaw composes onto the candidate quaternion without mutating the base

## What was actually built (deviations)

- **Linear slider** (`yaw-slider`, -180° to 180°) wired to `viewport.applyYaw(deg)`
- **Snap button** snaps to nearest of `[0, 45, 90, 135, 180, 225, 270, 315]` — NOT
  geometry-aware. This is a placeholder for the rotating-calipers snap.
- **`computeDefaultYaw`** exists in `compute.ts:187` and computes bbox-minimizing yaw
  via rotating calipers — it's used as the base orientation, but the snap button does
  not call it.
- **Export yaw**: main.ts:274-282 composes `qYaw * candidateQuat` for the exported STL,
  so yaw is correctly applied to the export.
