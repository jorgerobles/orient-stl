<objective>
Wire the H11 shadowed-overhang metric (resin suction-cup risk) into the composite score. Today `shadowed_overhang_fraction` is computed per-candidate in `score_all_directions` (lib.rs:281, args 32/0.02), travels in the 13-per-candidate metrics array at offset 9, and is DISPLAYED in the UI (CandidateList.ts:25) — but it does NOT affect ranking: `ScoreWeights` has no `w_shadowed`, `score_components`/`compute_norm_bounds` omit it, and `rank_by_weights_with_bounds` doesn't normalize or weight it. The metric is shown but never scored.

This is the highest resin-impact pending item: a candidate can rank #1 on overhang while creating a sealed cavity that traps resin (suction-cup failure). Wiring it as a 6th cost component (higher=worse, like overhang/footprint/cross/height — NOT inverted like surface) fixes that.

This is an API-breaking change across the WASM boundary (3 exports change shape): rank_candidates weights 5→6, compute_norm_bounds 10→12 floats, score_direction 8→9 floats. A stale WASM or a missed JS update = silent runtime breakage, so `make wasm` + `make type-check` is the hard gate.

Decisions (locked): (1) shadowed wired into ALL 3 rankers (weights, consensus, topsis) — no half-measures; (2) grid_res=32, tol_frac=0.02 as constants in scoring.rs (matching existing call sites), independent of cross_bins.

Output: 6-component scoring end-to-end, Rust + TS + profiles + tests, WASM rebuilt.
</objective>
