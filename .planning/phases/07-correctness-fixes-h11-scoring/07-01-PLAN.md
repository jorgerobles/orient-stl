<objective>
Fix a correctness bug in `refine_once` (core/src/lib.rs:478-482): the hill-climb perturbation vector is not perpendicular to the current best direction. Algebra confirms `dot(best_dir, perp) = best_dir[0]·best_dir[2]·(u2-u1) ≠ 0` in general — the three components do not derive from a single `b` in `dir × b`, so the "perpendicular" vector carries a spurious radial component that biases the stochastic search (worst for tilted directions like (0.7,0,0.7); zero only at pure poles where dir[0]·dir[2]=0).

The correct pattern already exists twice in the codebase: `scoring::perpendicular_basis` (scoring.rs:4) and `stability::find_perpendicular` (stability.rs:84) — both pick a non-parallel reference axis, take `cross(dir, ref)` normalized as e1, then `cross(dir, e1)` as e2. Reuse `perpendicular_basis` here.

Impact: the hill-climb only accepts improvements (never worsens), so this is a quality bug, not a crash — but it converges to worse local optima and inflates cross-seed variance (H7 refine-stability). Safe, isolated fix.

Output: a tested `tangent_perturbation` helper, `perpendicular_basis` made `pub(crate)`, `refine_once` rewired to use it, WASM rebuilt.
</objective>
