<objective>
Add winding normalization to the repair pipeline. After removing duplicate triangles, detect triangles whose face normal points inward (toward the mesh centroid rather than outward) and flip their vertex winding so the normal points outward.

This fixes speckled red overhang coloring on STLs with inconsistent winding, like `broken.stl` (~5% flipped normal headers from the break script). The winding check is based on vertex geometry (cross product), not STL header normals, so it correctly identifies and fixes inverted triangles regardless of the source.

Constraint: must work on triangle soup (no adjacency information). Using the centroid heuristic (dot product of face normal with centroid-to-triangle-center vector).
</objective>

## Tasks

1. **RED** — Write failing tests in `repair.rs`:
   - `normalize_winding_single_triangle`: single triangle, no flip needed
   - `normalize_winding_inverted_triangle`: single triangle with two vertices swapped → normal should be flipped back
   - `normalize_winding_two_triangles`: two opposite triangles (a thin shell), both should point outward
   - `normalize_winding_no_crash_on_degenerate`: zero-area triangle should not panic/skip
   - `normalize_winding_empty`: empty input → 0 flips
   - `normalize_winding_already_correct`: all normals already outward → 0 flips

2. **GREEN** — Implement `normalize_winding`:
   - Compute mesh centroid = average of all vertex positions
   - For each triangle, compute face normal via cross product, triangle center, and dot with centroid-ray
   - If dot < 0, swap two vertices (v2 ↔ v3) to flip winding; increment flip counter
   - Skip triangles where cross product is near zero (degenerate)

3. **REFACTOR** — Wire into `prepare_data_native`:
   - Add `repair::normalize_winding(&mut flat)` call after `repair::repair_mesh(&mut flat)`
   - Verify cargo test passes (89 tests → 96 tests)
