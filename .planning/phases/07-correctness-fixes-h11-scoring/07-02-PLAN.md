<objective>
Fix a correctness bug in `check_stability` (core/src/stability.rs:52-59): the center of mass is computed as the raw vertex centroid (Σ v / n), which biases toward finely-triangulated regions rather than toward actual mass. For resin miniatures with uneven triangulation (detailed hands/faces vs. flat cloak areas), this drifts the COM projection and can flip the `stable` verdict or distort the `margin` — a wrong go/no-go on tipping.

Replace with the area-weighted triangle-centroid average: `Σ(area_i · centroid_i) / Σ(area_i)`. This eliminates the triangulation-density bias (the dominant error). Honest caveat: this is still a surface-centroid approximation, not the true volume centroid — for thin resin shells they're close, for chunky solids they differ. It is a large improvement over the vertex centroid, not "exact COM".

Also clean up the unused `hull: &ConvexHull` parameter (`let _ = hull;` at line 76) — remove it and update all call sites. check_stability is internal (not `#[wasm_bindgen]`), so this is a safe signature change.

Output: tested `center_of_mass` helper, rewired `check_stability`, cleaned signature, WASM rebuilt.
</objective>
