# Plan 03.5-01: H5/H6 Scoring Backfill (Sub-plan 1 of 2)

**Phase:** 03.5 (scoring-expansion)
**Plan:** 01 of 02
**Status:** ✅ Complete (backfilled 2026-07-13)
**Mode:** TDD (Red-Green-Refactor)

## Goal

Add the two heuristics discovered in the PrusaSlicer comparison (H5 surface
quality, H6 print height) to orient-stl, and extend both rankers so all five
heuristics participate in the composite score.

## Tasks

### Task 1 — H5: `misalignmentScore` (TS) + `misalignment_score` (Rust)

Port of PrusaSlicer's `get_misalginment_score` (Rotfinder.cpp:88). For each
face, sum the L1 norm of the normal in the orientation frame `(dn, e1, e2)`:
`area × (|n·dn| + |n·e1| + |n·e2|)`. HIGHER = better (PrusaSlicer maximises
this to discourage axis-aligned flat shelves/walls).

**TDD:**
- RED: tests for face-on minimum, diagonal-direction higher, zero/empty edge cases
- GREEN: implement `misalignmentScore` in `compute.ts` and `misalignment_score` in `scoring.rs`

### Task 2 — H6: `min_z_height` (Rust) + wire existing `computeHeight` (TS)

PrusaSlicer's `find_min_z_height_rotation` minimises the rotated bbox Z size.
The TS side already had `computeHeight` (compute.ts:240) computing the vertex
extent along `dir`, stored as `estHeight` on `Candidate` — but it was only used
as a multiplicative fudge in consensus. Rust had no equivalent.

**TDD:**
- RED: tests for flat-slab-zero, two-layer-box, empty-mesh
- GREEN: implement `min_z_height` in `scoring.rs`; TS already had the function

### Task 3 — Extend `ScoreComponents` + `Candidate` + `SliceResult`

Add `surfaceQuality` (H5, maximise) and surface `height` (H6) to the data
model so the rankers can use them.

- `Candidate` gains `surfaceQuality: number`
- `SliceResult` gains `surface: number`
- `ScoreComponents` gains `surface_quality: f32` + `height: f32`
- `computeSlice` populates `surface` via `misalignmentScore`
- `mergeCandidates` passes `surfaceQuality` through

### Task 4 — Extend `ScoreWeights` + `WEIGHT_PRESETS`

- `ScoreWeights` gains `wSurface: number` + `wHeight: number`
- `WEIGHT_PRESETS` adds `surface-only (finish)` and `height-only (fast)`
- All existing presets extended with the two new fields (0 where irrelevant)

### Task 5 — Rewrite `rankByConsensus` + extend `rankByWeights`

Both rankers must include all five heuristics.

- `rankByWeights`: normalise surfaceQuality as cost (invert: `(max - val)/span`); normalise height as cost directly; add `wSurface*sn + wHeight*hn` to the weighted sum
- `rankByConsensus`: replace the `overhang × (1 + hN×0.5)` fudge with height as its own cost term in the `max()`; invert surfaceQuality to cost form; `1 - max(oN, fN, cN, sN, qN, hN)`

### Task 6 — UI tooltip + harness updates

- `main.ts` info tooltip gains `q:` (surface quality) readout
- `harness.rs` `WeightCfg` gains `w_surface` + `w_height`; new configs added; `invert_normalise` helper for the maximise metric
