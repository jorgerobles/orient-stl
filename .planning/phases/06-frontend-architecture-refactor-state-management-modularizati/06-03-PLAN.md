## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User file → AppController | File input crosses into controller lifecycle via FileDrop |
| Worker → AppController | Worker results cross thread boundary via postMessage |
| localStorage ↔ AppController | Config persistence crosses storage boundary on save/load |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06 | Tampering | Worker message shape (orient.worker.ts ↔ AppController) | mitigate | WorkerMessage discriminated union applied atomically to both worker postMessage and handler. Type narrowing via msg.type switch prevents shape confusion. Pitfall 2. |
| T-06-07 | Tampering | innerHTML in CandidateList/ScorePanel | mitigate | Only numbers and app-controlled strings (metric names, profile labels) reach innerHTML. File names are NEVER interpolated into HTML — stlName is used only for link.download (safe DOM property, not innerHTML). Preserve this invariant in the refactor. |
| T-06-08 | Tampering | localStorage schema in AppController.saveConfig/loadConfig | mitigate | Preserve SCHEMA_VERSION check in loadConfig. If version mismatches, removeItem and keep defaults. |
| T-06-09 | Information Disclosure | Observer subscription leak (Pitfall 4) | mitigate | AppController tracks all view subscriptions. View dispose() calls unsubscribe. On file reload, views are NOT recreated (preferred) — they re-render via existing subscriptions. |
| T-06-SC | Tampering | npm install | N/A | Zero packages installed this plan. |
